#### **3. Opening and Initializing the Report File**

After successfully creating the report file:

- Click on the newly created file to **open it**.
- At the **bottom-right corner** of the screen, locate the **Start icon button**.
- Click the **Start button** to initiate the configuration of the report file.

#### **4. Enabling Edit Mode**

To modify the contents of the file:

- Click on the **Edit icon button**, typically represented with a pencil or similar graphic.
- This activates edit mode, allowing structural and content-level modifications.

#### **5. Accessing the Widget Sidebar**

For enhanced customization and content elements:

- In the **upper-right corner** of the interface, you will see **three icon buttons**.

#### 6. Adding a Table Widget

From the widget sidebar:

- Locate and select the **‚ÄúTables Widget‚Äù**.
- Once selected, various options will appear.

#### 7. Configuring the Table Widget ‚Äì Header Properties

The first section visible is **Widget Header Properties**.

**Header Properties Include:**

- **Always Display Header**: Toggle to always show the widget header.
- **Title Text**: Add a title to your table widget.
- **Font Size, Weight, Alignment**: Customize the appearance of your title.
- **Font Color**: Select the appropriate text color using the color picker.
- **Additional Properties**:
  - **Start and End Date** selectors
  - **Left and Right Logo Upload**: Upload logos or images on either side of the title by clicking ‚ÄúChoose File‚Äù.

Upon clicking on the **table area**, a new panel titled **Table Properties** appears to the right.

**Editable Table Area:**

#### 11. Advanced Configuration ‚Äì Styling and Conditional Formatting

**Data Formatting Options**

Under the **Data Formatting** section, users can define how numeric or time-based data should be presented.

There are three primary options in the dropdown:

1. **Decimal Point**
   - Allows users to specify the number of decimal places shown in numerical values.
2. **Timestamp**
   - Used when displaying time-based data.
   - Upon selection, a secondary dropdown **‚ÄúSelect Timestamp Format‚Äù** appears.
   - Choose from available standard formats such as `YYYY-MM-DD HH:mm:ss` or `MM/DD/YYYY`.
   - This ensures consistent and localized representation of time values.

### Static Table Configuration

The Static Table allows fixed-value display with cell-specific formatting, aggregation, and filters. Since many features are identical to the Table Widget, only distinct or additional configurations are explained here.

### **Selecting the Static Table**

Select **‚ÄúStatic Table‚Äù** from the available table types under the **Table Group** panel.

![Stat1 Pn](/images/stat1.png)

### **Table Properties** 

After opening the Static Table, you'll find that **general table properties** (e.g., Text Formatting, table source columns, Grouping, Aggregation, Sorting  ) mirror those of the Table Widget.

However, individual **cell-level settings** differ and are described below.

---

### **Configuring Individual Cells** 

When you click on a cell in the table grid, **cell-specific configuration options** will appear on the right-hand panel.

### Cell 1: Time Column Configuration

- **Column**: Select the column name (e.g., `time`) from the dropdown.
- **Aggregation**: **Mandatory** ‚Äî choose a suitable function (e.g., `Max`, `Min`) for timestamp-related columns.
- **Filter**: Not required in this case.
- **Data Formatting**: Same as in Table Widget; select **Timestamp** format.
  - Example: `MM/dd/yyyy HH:mm`

    ![Stat1 2 Pn](/images/stat1.2.png)

---

### **Cell 2: Data Column Configuration**

- **Column**: Select the desired tag from the dropdown.
- **Aggregation**: Choose based on your requirement (e.g., `Min`, `Avg`).
- **Filter**: **Optional** ‚Äî can be applied to show data conditionally.
  - Example: `table1_tag1_max >= 100`
- **Data Formatting**: Select **Decimal Point** with the required precision (e.g., `2`).
- **Cell Color**: Use conditional color formatting.
  - Example: `{table1.tag1} > 400` ‚Üí Color: Red
